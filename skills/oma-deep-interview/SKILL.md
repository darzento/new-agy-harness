# OMA Deep Interview

이 스킬은 "무엇을 만들지"가 확실치 않을 때, 작업을 바로 시작하지 않고 사용자와의 심층 대화를 통해 시스템의 경계, 제약사항, 숨겨진 기획 의도를 명확하게 끌어냅니다.

## When to use this skill
- 사용자가 요구사항을 매우 짧고 모호하게 던질 때 (예: "TODO 앱 하나 만들어줘", "게시판 설계해줘").
- 사용자가 "딥 인터뷰", "요구사항 도출", "기획 회의하자" 등의 키워드를 사용할 때.

## Instructions
1. 절대 먼저 코드를 작성하거나 설계도를 완성하지 않습니다.
2. 한 번의 턴에 너무 많은 질문을 쏟아내지 말고, 핵심적인 1~2개의 질문만 던져 대화를 이어갑니다.
3. 사용자의 답변이 충분히 모여 "기획안(PRD)을 작성할 수 있는 임계점"에 도달했는지 스스로 판단합니다.
4. 기획안은 "_workspace" 폴더에 저장합니다.

## Workflow
1. **Initial Probing**: 사용자의 첫 요구사항을 분석하여 가장 크고 중요한 "가정"이 무엇인지 식별하고 첫 번째 핵심 질문을 던집니다.
2. **Iterative Questioning**: 사용자의 답변을 듣고, 시스템의 비기능적 요구사항(성능, 보안 등)이나 엣지 케이스에 대한 꼬리 질문을 던집니다.
3. **Requirement Synthesis**: 대화가 3~4턴 이상 진행되어 명확성이 확보되면, 도출된 요구사항을 바탕으로 `omc-planner`에게 넘길 수 있는 **"최종 정리된 요구사항 명세 텍스트"**를 산출하고 인터뷰를 종료합니다. 이후 `oma-team` 워크플로우로 매끄럽게 연결합니다.
